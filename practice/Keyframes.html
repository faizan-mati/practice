CSS Keyframes & Animations

Complete Guide with Real-World Examples

What are CSS Keyframes?

CSS Keyframes allow you to create smooth animations by defining specific points during an animation sequence. Think of keyframes as "snapshots" of how an element should look at different points in time during an animation.

Basic Syntax

@keyframes animationName { 0% { /* Starting state */ property: value; } 50% { /* Midpoint state */ property: value; } 100% { /* Ending state */ property: value; } } /* Apply animation to element */ .element { animation: animationName duration timing-function iteration-count; }

@keyframes Rule

The @keyframes rule defines the animation code. You can use percentages (0% to 100%) or keywords (from, to) to specify when each change should happen.

Example: Bouncing Ball

@keyframes bounce { 0%, 100% { transform: translateY(0); } 50% { transform: translateY(-50px); } } .bounce-ball { animation: bounce 2s ease-in-out infinite; }

animation-name Property

The animation-name property specifies which keyframe animation to use. It must match the name defined in @keyframes.

Example: Loading Spinner

@keyframes spin { from { transform: rotate(0deg); } to { transform: rotate(360deg); } } .spinner { animation-name: spin; animation-duration: 1s; animation-timing-function: linear; animation-iteration-count: infinite; }

animation-iteration-count Property

Controls how many times the animation should repeat. Values can be a number, 'infinite', or 'initial'.

Once (1)

Plays once then stops

Three Times (3)

Repeats 3 times

Infinite

Never stops repeating

.once { animation-iteration-count: 1; } .three-times { animation-iteration-count: 3; } .infinite { animation-iteration-count: infinite; }

All Animation Properties

/* Individual Properties */ animation-name: myAnimation; animation-duration: 2s; animation-timing-function: ease-in-out; animation-delay: 0.5s; animation-iteration-count: infinite; animation-direction: alternate; animation-fill-mode: forwards; animation-play-state: running; /* Shorthand Property */ animation: myAnimation 2s ease-in-out 0.5s infinite alternate forwards;

Property Explanations:

  • animation-duration: How long the animation takes (e.g., 2s, 500ms)
  • animation-timing-function: Speed curve (ease, linear, ease-in, ease-out, ease-in-out)
  • animation-delay: Delay before animation starts
  • animation-direction: Direction of animation (normal, reverse, alternate, alternate-reverse)
  • animation-fill-mode: What happens before/after animation (none, forwards, backwards, both)
  • animation-play-state: Whether animation is running or paused

Real-World Examples

1. Button Pulse Effect

@keyframes pulse { 0% { box-shadow: 0 0 0 0 rgba(78, 205, 196, 0.7); } 70% { box-shadow: 0 0 0 20px rgba(78, 205, 196, 0); } 100% { box-shadow: 0 0 0 0 rgba(78, 205, 196, 0); } } .pulse-button { animation: pulse 2s ease-in-out infinite; }

2. Sliding Card Animation

Sliding Content

This card slides in from the left, stays visible, then slides out to the right!

@keyframes slideIn { 0% { transform: translateX(-100%); opacity: 0; } 20% { transform: translateX(0); opacity: 1; } 80% { transform: translateX(0); opacity: 1; } 100% { transform: translateX(100%); opacity: 0; } }

3. Complex Multi-Step Animation

@keyframes complexMove { 0% { transform: translateX(0) rotate(0deg); background: #ff6b6b; border-radius: 0; } 25% { transform: translateX(200px) rotate(90deg); background: #4ecdc4; border-radius: 25%; } 50% { transform: translateX(200px) translateY(-100px) rotate(180deg); background: #45b7aa; border-radius: 50%; } 75% { transform: translateX(0) translateY(-100px) rotate(270deg); background: #96ceb4; border-radius: 25%; } 100% { transform: translateX(0) translateY(0) rotate(360deg); background: #ff6b6b; border-radius: 0; } }

4. Typing Text Effect

This text appears like it's being typed!
@keyframes typing { from { width: 0; } to { width: 100%; } } @keyframes blink-cursor { from, to { border-color: transparent; } 50% { border-color: #4ecdc4; } } .typing-text { white-space: nowrap; overflow: hidden; border-right: 3px solid #4ecdc4; animation: typing 4s steps(40, end), blink-cursor 0.75s step-end infinite; }

5. Floating Card with Hover

Hover Over Me!

I float gently, but when you hover, I pause and lift up more!

@keyframes float { 0%, 100% { transform: translateY(0px); } 50% { transform: translateY(-10px); } } .hover-card { animation: float 6s ease-in-out infinite; transition: transform 0.3s ease; } .hover-card:hover { transform: translateY(-10px) scale(1.02); animation-play-state: paused; }

Animation Timing Functions

Timing functions control the speed curve of animations, making them feel more natural.

Ease

Starts slow, speeds up, then slows down

Linear

Constant speed throughout

Ease-in-out

Slow start and end, fast middle

/* Built-in timing functions */ ease /* Default: slow start, fast middle, slow end */ linear /* Constant speed */ ease-in /* Slow start */ ease-out /* Slow end */ ease-in-out /* Slow start and end */ /* Custom cubic-bezier */ cubic-bezier(0.25, 0.1, 0.25, 1.0) /* Step functions */ steps(4, end) /* Discrete steps */

Performance Tips

Best Practices for Smooth Animations

  • Use transform and opacity: These properties are GPU-accelerated
  • Avoid animating layout properties: width, height, padding, margin cause reflows
  • Use will-change: Hint to browser about upcoming animations
  • Keep animations under 60fps: Aim for smooth 60fps performance
  • Use animation-fill-mode: Control element state before/after animation
/* Good for performance */ .good-animation { will-change: transform; animation: moveAndFade 1s ease-out; } @keyframes moveAndFade { from { transform: translateX(-100px); opacity: 0; } to { transform: translateX(0); opacity: 1; } } /* Avoid animating these properties */ .avoid { /* Don't animate: width, height, padding, margin, left, top */ }
