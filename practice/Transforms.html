Transforms are applied from right to left (or in the order written). rotate(45deg) translateX(100px) is different from translateX(100px) rotate(45deg).

‚ö° Performance Best Practices

‚úÖ Good for Performance

  • transform: translateX()
  • transform: scale()
  • transform: rotate()
  • opacity changes

Why: These properties are handled by the GPU and don't trigger layout or paint.

‚ùå Avoid for Animations

  • left, top, right, bottom
  • width, height
  • margin, padding
  • border-width

Why: These properties trigger layout recalculations and are expensive.

üí° Performance Tips

  • Always use will-change: transform; for elements you plan to animate
  • Use transform3d() or translateZ(0) to trigger hardware acceleration
  • Remove will-change after animations complete
  • Prefer transforms over changing layout properties

üîß Browser Support & Advanced Tips

Browser Support

CSS transforms have excellent browser support. For older browsers, you might need vendor prefixes:

-webkit-transform: rotate(45deg); /* Safari, Chrome */ -moz-transform: rotate(45deg); /* Firefox */ -ms-transform: rotate(45deg); /* IE 9 */ transform: rotate(45deg); /* Standard */

üé® Advanced Transform Properties

  • transform-origin: Change the pivot point for transforms
  • transform-style: preserve-3d: Enable 3D transforms
  • perspective: Add 3D depth to transforms
  • backface-visibility: Control visibility of flipped elements
