Master d-none, d-md-block, order-md-1 and more responsive utilities

📱 Display Utilities (d-*)

Basic Display Classes

.d-none → display: none .d-block → display: block .d-inline → display: inline .d-inline-block → display: inline-block .d-flex → display: flex .d-grid → display: grid
d-block: Always visible as block element
d-none d-md-block: Hidden on mobile (XS, SM), visible from MD and up
d-block d-lg-none: Visible on mobile/tablet, hidden on large screens

Responsive Display Examples

d-none d-sm-block d-md-none: Only visible on SM breakpoint
Multiple breakpoints: Flex on MD, Grid on LG, Inline-block on XL

🔄 Flex Order Utilities (order-*)

Order Classes

.order-first → order: -1 (first) .order-0 → order: 0 .order-1 → order: 1 .order-2 → order: 2 .order-3 → order: 3 .order-4 → order: 4 .order-5 → order: 5 .order-last → order: 13 (last) /* Responsive variants */ .order-sm-1, .order-md-1, .order-lg-1, .order-xl-1, .order-xxl-1

✅ Best Practices & Tips

✅ DO
  • Use d-none d-md-block for desktop-only content
  • Combine multiple breakpoint classes for fine control
  • Test on actual devices, not just browser resize
  • Use order classes to improve mobile UX
  • Consider content hierarchy on different screen sizes
❌ DON'T
  • Hide important content on mobile without alternatives
  • Over-complicate with too many breakpoint variations
  • Forget to test middle breakpoints (SM, MD)
  • Use display utilities for animations (use CSS transitions)
  • Rely solely on hiding/showing - consider reorganizing content
