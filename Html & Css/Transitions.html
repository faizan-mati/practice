🚀 What are CSS Transitions?

CSS transitions allow you to change property values smoothly over a given duration. They provide a way to control animation speed when changing CSS properties, creating smooth animations between different states of an element.

💡 Key Benefit: Transitions make your website feel more interactive and professional by adding smooth animations without JavaScript.
selector { transition: property duration timing-function delay; }

🎯 Basic Transition Example

Let's start with a simple example. Hover over the square below:

This square changes color, size, and shape on hover!

.basic-hover { width: 100px; height: 100px; background: #ff6b6b; transition: all 0.3s ease-in-out; } .basic-hover:hover { background: #4ecdc4; transform: scale(1.1); border-radius: 50%; }

🔧 Transition Properties Breakdown

1. transition-property

Specifies which CSS properties should be transitioned. You can target specific properties or use 'all'.

Color Only

transition-property: background-color;

Transform Only

transition-property: transform;

Opacity Only

transition-property: opacity;

/* Target specific properties */ transition-property: background-color, transform, opacity; /* Target all animatable properties */ transition-property: all; /* No transitions */ transition-property: none;

2. transition-duration

Sets how long the transition should take to complete. Hover over these elements to see different speeds:

Fast (0.1s)

Quick and snappy

Medium (0.5s)

Balanced timing

Slow (2s)

Dramatic effect

transition-duration: 0.3s; /* Seconds */ transition-duration: 300ms; /* Milliseconds */ transition-duration: 0.3s, 0.5s; /* Multiple values */

3. transition-timing-function (Easing)

Controls the speed curve of the transition. Hover over the container below to see different easing functions in action:

Different Easing Functions

linear
ease (default)
ease-in
ease-out
ease-in-out
cubic-bezier (bounce)

Hover to see the animations!

transition-timing-function: linear; /* Constant speed */ transition-timing-function: ease; /* Slow start, fast, slow end */ transition-timing-function: ease-in; /* Slow start */ transition-timing-function: ease-out; /* Slow end */ transition-timing-function: ease-in-out; /* Slow start and end */ transition-timing-function: cubic-bezier(0.68, -0.55, 0.265, 1.55); /* Custom curve */
🎨 Pro Tip: Use ease-in-out for most UI interactions as it feels the most natural. Use ease-out for elements entering the screen and ease-in for elements leaving.

✅ Best Practices & Tips

🎯 Performance Tips:

  • Prefer animating transform and opacity for better performance
  • Avoid animating width, height, top, left as they trigger layout
  • Use will-change property for complex animations
  • Keep transitions under 300ms for UI interactions

⚠️ Common Mistakes:

  • Making transitions too long (over 500ms) for simple interactions
  • Animating too many properties with all
  • Not considering accessibility - some users prefer reduced motion
  • Forgetting to test on mobile devices

Accessibility Consideration

/* Respect user's motion preferences */ @media (prefers-reduced-motion: reduce) { * { transition: none !important; } }

Complete Syntax Reference

/* Individual properties */ transition-property: background-color, transform; transition-duration: 0.3s, 0.5s; transition-timing-function: ease-in-out, ease; transition-delay: 0s, 0.1s; /* Shorthand syntax */ transition: background-color 0.3s ease-in-out 0s, transform 0.5s ease 0.1s; /* Simple shorthand */ transition: all 0.3s ease-in-out;
