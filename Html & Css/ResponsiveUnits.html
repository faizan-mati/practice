REM Units (Root EM)

REM is relative to the root element's font size (usually 16px). It provides consistency across components.

Key Points:
• 1rem = root font size (typically 16px)
• Not affected by parent element font size
• Perfect for consistent spacing and typography
• Respects user's browser font size preferences
html { font-size: 16px; } /* Default root size */ .small-text { font-size: 0.75rem; } /* 12px */ .normal-text { font-size: 1rem; } /* 16px */ .large-text { font-size: 1.5rem; } /* 24px */ .xlarge-text { font-size: 2rem; } /* 32px */
0.75rem (Small Text)
1rem (Normal Text)
1.5rem (Large Text)
2rem (XLarge Text)

Best Use Cases: Main typography, consistent spacing (margins, padding), component sizing, layout dimensions.

EM Units (Relative to Parent)

EM is relative to the parent element's font size. It creates proportional scaling within components.

Key Points:
• Relative to immediate parent's font size
• Creates cascading/compound effects
• Great for component-level scaling
• Can create unexpected results when nested
.parent { font-size: 20px; } .child1 { font-size: 0.8em; } /* 16px (20px × 0.8) */ .child2 { font-size: 1.2em; } /* 24px (20px × 1.2) */ /* Nested EM compounds */ .nested { font-size: 1.5em; } /* 30px */ .nested .nested-child { font-size: 0.8em; } /* 24px (30px × 0.8) */

Parent font-size: 20px

0.8em = 16px
1.2em = 24px
1.5em = 30px
Nested: 0.8em = 24px

Best Use Cases: Button padding relative to text size, icon sizing, dropdown menus, component internal spacing.

Viewport Units (vw, vh, vmin, vmax)

Viewport units are relative to the browser's viewport dimensions, perfect for full-screen layouts.

Viewport Unit Types:
• vw: 1% of viewport width
• vh: 1% of viewport height
• vmin: 1% of viewport's smaller dimension
• vmax: 1% of viewport's larger dimension
/* Viewport Width Examples */ .half-width { width: 50vw; } /* 50% of viewport width */ .full-width { width: 100vw; } /* Full viewport width */ /* Viewport Height Examples */ .hero-section { height: 100vh; } /* Full viewport height */ .quarter-height { height: 25vh; } /* Quarter viewport height */ /* Responsive Typography */ .responsive-title { font-size: 5vw; } /* Scales with width */
50vw width (resize browser to see change)
30vh height (scroll or resize to see effect)
Unit Calculation Use Case
10vw 10% of viewport width Responsive containers
50vh 50% of viewport height Half-screen sections
5vmin 5% of smaller dimension Square elements
10vmax 10% of larger dimension Consistent sizing

Best Use Cases: Hero sections, full-screen layouts, responsive typography, modal overlays, responsive images.

CSS clamp() Function

clamp() sets a minimum, preferred, and maximum value in one declaration. It's the ultimate responsive tool.

Syntax: clamp(minimum, preferred, maximum)
• minimum: Smallest allowed value
• preferred: Ideal value (usually responsive)
• maximum: Largest allowed value
/* Typography that scales but stays readable */ .heading { font-size: clamp(1.5rem, 4vw, 3rem); } /* min: 24px, scales with width, max: 48px */ /* Responsive padding */ .container { padding: clamp(1rem, 5vw, 3rem); } /* Responsive width */ .content { width: clamp(300px, 80vw, 800px); }

This heading uses clamp(1.5rem, 4vw, 3rem)

This text uses clamp(0.9rem, 2.5vw, 1.2rem) and this container uses clamp(300px, 80vw, 800px) for width.

Resize your browser to see how everything scales smoothly between minimum and maximum values!

Common clamp() Patterns:

/* Responsive Typography Scale */ .h1 { font-size: clamp(2rem, 5vw, 4rem); } .h2 { font-size: clamp(1.5rem, 4vw, 3rem); } .h3 { font-size: clamp(1.25rem, 3vw, 2rem); } .body { font-size: clamp(1rem, 2.5vw, 1.2rem); } /* Responsive Spacing */ .section { margin-bottom: clamp(2rem, 8vh, 6rem); } .card { padding: clamp(1rem, 4vw, 2.5rem); } /* Responsive Layouts */ .sidebar { width: clamp(250px, 25vw, 400px); } .container { max-width: clamp(320px, 90vw, 1200px); }

Best Use Cases: Responsive typography, fluid spacing, container sizing, any value that needs min/max constraints.

2. Best Practices Summary

Unit Best For Avoid For
rem Typography, consistent spacing, component sizes Viewport-based layouts
em Component internal spacing, icon sizes Complex nested structures
vw/vh Full-screen layouts, hero sections Small UI elements, body text
clamp() Responsive everything with boundaries Simple fixed layouts
Pro Tips:
• Combine units: use clamp() with vw/vh for perfect responsiveness
• Set a base font-size on html for consistent rem scaling
• Use em for component internals, rem for layout
• Always test on multiple screen sizes
• Consider accessibility - respect user font size preferences
